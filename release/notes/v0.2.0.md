# Agent Lifecycle Kit v0.2.0

This release adds provider-neutral model routing and makes model-backed task
attempts enforceable through host-attested usage receipts.

## Highlights

- Added deterministic provider-neutral model routing for lifecycle phases and
  task attempts.
- Added host-local model profile validation while keeping concrete provider
  model names out of portable core contracts.
- Added workflow enforcement for model-backed task attempts: `workflow
  task-result` now requires a host-attested usage receipt bound to the run,
  task, attempt, plan digest, source revision and route decision digest.
- Added fail-closed protection against critical review downgrades to `budget`
  or `local-compact` model classes.
- Updated adapter descriptors and the offline adapter baseline with
  workflow-enforced model-route execution semantics.

## Adapter status

Codex, Claude Code, Cursor, Hermes and OpenCode adapters remain
`EXPERIMENTAL`. They declare the fail-closed model-route execution contract, but
`VERIFIED` status still requires live host evidence with usage receipts and
quality/cost results.

## Install

Codex:

```bash
codex plugin marketplace add avksp/agent-lifecycle-kit --ref v0.2.0
codex plugin add agent-lifecycle-kit@agent-lifecycle-kit
```

Hermes:

```bash
for skill in agent-first-planning audit-agent-plan agent-plan-to-workers agent-workflow-orchestrator audit-plan-implementation; do
  hermes skills install "https://raw.githubusercontent.com/avksp/agent-lifecycle-kit/v0.2.0/skills/${skill}/SKILL.md"
done
```

OpenCode source projection:

```bash
KIT=/path/to/agent-lifecycle-kit
mkdir -p .opencode/plugins
cp "$KIT"/opencode.json ./opencode.json
cp "$KIT"/adapters/opencode/plugins/agent-lifecycle-kit.js .opencode/plugins/
```
